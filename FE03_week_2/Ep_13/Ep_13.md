### Why do we need test cases?
- because we need maintainibility
- also to check that   
    - e.g 
        - we have an app, and we have a feature we wanna introduce, but we do not want to break our app 
        - how share are we that we are not breaking the app? unit test gives us that confidence 


### Test driven development
- we write test cases before we develop our app
- this is very good approach, but it slows down very much, the development process 

### Different types of testing
- manual testing
    - requires humans for testing
- automated testing
    - a code to testing a code
    - selenium testing 
        - entire workflow of testcases 
- end to end testing 
    - cypress
        - will do a testing like how a user would do testing 
- Unit testing
    - testing small parts of the app 
- Integration testing 
- regression testing
- smoke testing 
- black ops testing 
- React testing library uses jest behind the scenes


### Testing steps
- install jest, react testing library 
- configure the jest
    - npx jest --init
        - we use npx bc we only need to do it once
- create first test file 
    - When we create folder like "__tests__" jest will consider this folder as testing files 
    - For writing the test file, there is convention to write ".test" after naming the file 
        - e.g sum.test.js
    - face error, cannot use import statement outside a module 

    - JSON and javascript object are not same 
    - Difference between json and javascript object 

- write tests and add test directory in gitignore 

- When you run npm run test, what happens is, it doesn't perform the test by running on browser
    - its running on different environment provided by jest 
    - the environment is jsdom in jest 
        - its not a browser, but it will have browser related functionalities 

### While writing test cases

- While writing test cases, write appropriate names for the test case   
    - "Logo" - not clear enough to understand the test case
    - "Logo should appear above on rendering" - clear 

- We cannot make use of react router in testing, so we make use of static router in .test.js files
    - import {StaticRouter} from "react-router-dom/server"
